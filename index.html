Commands

  move: command[|move]

  command:
    F(x,y)                         //fell trees
    C(x,y)                         //cut peat
    B(id,x,y)                      //build
    U(id)[*]                       //use (with prior, add *)
    U(id,resources)[*]             //use (with resources as input) (with prior, add *)
    U(id,resources1,resources2)[*] //use (with resources as input) (with prior, add *)
    U(id,x1,y1[,x2,y2[,]])[*]      //use (on the tiles on your landscape at the listed coordinates)
    W(color,payment)               //work contract a player... basically it just pays them
    V(resources)                   //convert grain, pennies, nickels, wine, and beer
    P(y,plotside)                  //buy a plot, place its upper left corner on the row y
    D(y,districtside)              //buy a district, place it on the row y
    S(id,x,y,resources)            //settle a settlement (e.g. S04) at (x,y), pay for it with resources.

  id: @## //the building id

  resources1: resources //usually the "inputs"

  resources2: resources //usually the "output", but sometimes a secondary action

Sometimes (a lot of the time) you need to do a lot of things in a turn, such as Workorder someone and then Use a building, or use a building which lets you use another building, such as the Calefactory which lets you fell trees AND cut peat. Rather than trying to figure out some sort of complicated nested nature of the U() command, the syntax just lets you string together many commands into a single move, for example:

W(WHITE,Wn)|U(F10,2,-1)|B(G39,2,-1)|U(G39,CoCo)*

This would work-contract White to do something, gifting him a wine (which is consumed). He would use the Carpentry (F10) to remove the forest at 2, -1 and then build the Estate there, and then use it with a prior (because it has an asterisk) and give it 2 coal.

You're able to do a lot of things that aren't really legal moves, such as build multiple times, or Fell-Trees your entire land in one go. The players have to police things like this. The game is a little optimistic, in that it assumes players don't cheat. Rather than code as though the exceptions are the legal things, I tried to code as though the exceptions are the illegal things. Some things are caught, such as building a landscape in an illegal spot. Most things aren't.

Coordinates

The upper left square of your original 5x2 landscape is 0,0. Positive X is right, Positive Y is down. Coordinates can go negative obviously, but 0,0 always stays the origin.

Buildings

Almost all buildings consume the "resources" for their 3rd parameter of the Use command.

The buildings that get you things from tokens on the wheel (Farmyard, Cloister Office, Quarry, etc) just take an input of the token. So for instance if you want to use your Clay Mound to take Clay, do U(LR1,Cl). If you want to take clay with the Joker, add Jo. The farmyard needs to know if you want sheep or grain, so if you want sheep, do U(LR2,Sh). If you want to use the farmyard to take sheep by using the joker, do U(LR2,ShJo) or U(LR2,JoSh). Order doesn't matter.

Printing Office is the only building that uses more than one coordinate. You feed it coordinates of every forest you want to remove in exchange for a book.

Some buildings take two resource parameters. They are:

  • Bakery
  • Cloister Courtyard
  • Cloister Library
  • Pilgrimage Site
  • Shipping Company
  • Winery
  • House of the Brotherhood

Using the Priory

To use the priory to use another building, just use the priory U(G01), then use another building. Right now the game doesn't check to make sure that the building you use with it has a prior on it.
